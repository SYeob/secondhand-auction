# Pa-Bi Auction QA 프로젝트 개요

## 1. 프로젝트 목적

Pa-Bi Auction 서비스를 대상으로 요구사항 분석, 위험 기반 테스트 설계, UI/API 자동화, 결함 관리 및 테스트 결과 보고 과정을 수행한다.

본 프로젝트의 목적은 단순히 Playwright 테스트 코드를 작성하는 것이 아니라, 경력직 QA 엔지니어에게 필요한 다음 역량을 하나의 GitHub 저장소에서 보여주는 것이다.

- 요구사항과 비즈니스 위험 분석
- 테스트 우선순위 및 범위 결정
- 핵심 테스트 시나리오와 케이스 설계
- 수동 테스트와 자동화 대상 구분
- 결함 재현 및 회귀 테스트 관리
- CI 환경에서의 반복 검증

---

## 2. 대상 기능

| 기능 | 설명 | 중요도 |
|---|---|---|
| 로그인 | 사용자 인증 및 보호 화면 접근 제어 | P1 |
| 상품 등록 | 경매 상품 정보와 기간 등록 | P1 |
| 상품 조회 | 경매 목록 및 상세 정보 조회 | P1 |
| 입찰 | 유효 금액 입찰 및 최고가 갱신 | P0 |
| 낙찰 및 경매 종료 | 종료 시점과 최고 입찰자 기준 낙찰 처리 | P0 |
| 마이페이지 | 등록·입찰·낙찰 내역 조회 | P1 |
| 좋아요 | 관심 상품 저장 및 조회 | P2 |

---

## 3. 핵심 품질 목표

1. 입찰 금액과 최고 입찰자가 정확하게 반영되어야 한다.
2. 경매 종료 시점 이후에는 추가 입찰이 허용되지 않아야 한다.
3. 낙찰 결과와 거래 정보는 권한이 있는 사용자에게만 제공되어야 한다.
4. 주요 기능 변경 후에도 핵심 사용자 흐름이 정상적으로 유지되어야 한다.

---

## 4. 테스트 범위

### 포함 범위

- 주요 기능의 정상·예외 흐름
- 입력값 검증
- 인증 및 권한 확인
- 경매 상태 전환
- UI 자동화
- 핵심 API 검증
- 회귀 테스트
- 기본 보안 점검

### 제외 범위

- 실제 결제 연동
- 실제 SMS 및 이메일 발송
- 대규모 성능·부하 테스트
- 운영 인프라 장애 복구
- 외부 OAuth 제공자 자체 검증

---

## 5. 기술 구성

- Frontend: React, Vite
- UI Automation: Playwright
- Language: TypeScript
- CI: GitHub Actions
- Report: Playwright HTML Report
- Version Control: GitHub

- 테스트 결과와 잔여 위험이 문서화됨
