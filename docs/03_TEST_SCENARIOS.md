# 테스트 시나리오

| ID           | 시나리오                           | 우선순위 | 구현 |
| ------------ | ---------------------------------- | -------- | ---- |
| UI-SMOKE-001 | 홈 핵심 요소 표시                  | P1       | 완료 |
| UI-AUTH-001  | 정상 로그인                        | P1       | 완료 |
| UI-AUTH-002  | 잘못된 비밀번호 로그인 실패        | P1       | 완료 |

| UI-ITEM-001  | 로그인 사용자 상품 등록 및 삭제    | P1       | 완료 |
| UI-BID-001   | 현재가보다 높은 입찰 성공          | P0       | 완료 |
| UI-BID-002   | 현재가와 같은 입찰 차단            | P0       | 완료 |
| UI-BID-003   | 현재가보다 낮은 입찰 차단          | P0       | 완료 |
| UI-BID-004   | 비로그인 입찰 차단                 | P0       | 완료 |
| UI-CLOSE-001 | 종료 상태 및 입찰 버튼 비활성화    | P0       | 완료 |
| UI-CLOSE-002 | 낙찰자 연락처 표시                 | P0       | 완료 |
| UI-CLOSE-003 | 비낙찰자 연락처 미노출             | P0       | 완료 |
| UI-MY-001    | 본인이 입찰한 상품 마이페이지 조회 | P1       | 완료 |

## 3. 수동·탐색 유지 시나리오

| ID           | 시나리오                           | 이유                         |
| ------------ | ---------------------------------- | ---------------------------- |
| MAN-UI-001   | 반응형 레이아웃과 오류 문구 이해도 | 시각·주관 평가 필요          |
| MAN-BID-001  | 입찰 버튼 초고속 반복 클릭         | 조작·네트워크 조건 탐색 필요 |
| MAN-TIME-001 | 서버와 클라이언트 시각 차이        | 환경 제어 필요               |
| MAN-SEC-001  | 브라우저 저장소·쿠키·헤더 점검     | 도구 기반 보안 점검          |

## 4. 실행 묶음

```powershell
# 전체
pytest -c tests/pytest.ini tests

# P0
pytest -c tests/pytest.ini tests -m p0

# UI 또는 API
pytest -c tests/pytest.ini tests -m ui
pytest -c tests/pytest.ini tests -m api
```
